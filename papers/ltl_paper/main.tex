\miniheading{Abstract}
We propose an assertional reinterpretation of the Functionality Cotyledon in A.\ Wayne Wymore's Tricotyledon Theory of System Design (T3SD). In classical T3SD, functional intent is verified constructively via a reference system $Z_{\text{spec}}$ and homomorphism $h: Z_{\text{impl}} \rightarrow Z_{\text{spec}}$. We define $\mathcal{FC}_{\mathcal{L}}(\Phi) = \{ Z \mid Z \models_{\mathcal{L}} \Phi \}$, where $\Phi$ is a set of temporal logic properties over Wymore discrete-system trajectories and $\mathcal{L}$ is a chosen dialect (e.g., propositional LTL, FO-LTL, STL). The framework is logic-agnosticsuch that expressivity is not restricted. Our contributions are (i) a parameterized assertional definition of Wymore's Functionality Cotyledon, (ii) trace semantics grounded in Wymore's $\text{STZ}$/$\text{OTZ}$ functions, and (iii) a verification paradigm comparison illustrated by a decentralized UAV swarm case study with continuous-valued state at discrete ticks.

\phantomsection
\subsubsection{Keywords}
Tricotyledon Theory of System Design (T3SD), Linear Temporal Logic (LTL), Formal Verification, Model-Based Systems Engineering (MBSE).

\section{Introduction}

Model-Based Systems Engineering (MBSE) promises improved traceability and earlier verification, yet many requirements remain expressed in natural language or in diagrammatic forms that lack formal semantics \parencite{salado2019mbse,INCOSE2023}. We contend that this gap limits automated validation: engineers cannot mechanically determine whether an implementation satisfies temporal intent without reconstructing informal arguments. A.\ Wayne Wymore's Tricotyledon Theory of System Design (T3SD) offers one rigorous alternative: defining functional intent through a constructive reference system and homomorphic verification \parencite{Wymore1993}. Recent work has investigated how Wymore's framework can underpin practical MBSE \parencite{salado2019wymore}.

In classical T3SD, functional requirements are formalized as a specification system $Z_{\text{spec}}$, and correctness of an implementation $Z_{\text{impl}}$ is established by finding a system homomorphism $h: Z_{\text{impl}} \rightarrow Z_{\text{spec}}$ (Figure~\ref{fig:comparison}A). Constructing and maintaining such morphisms is manual and poorly matched to evolving implementations, especially when concurrent subsystems induce large proof obligations \parencite{Wymore1993}. Consider the case of large scale swarms of unmanned systems: when a single swarm element executes Return to Base (RTB) and leaves the swarm, total system participation changes. Maintaining universal constructive proofs across participation transitions can require substantial verification even though mission intent may still be satisfiable overall.

We propose an assertional alternative: encode functional intent as a property set $\Phi$ in a temporal logic dialect $\mathcal{L}$ and verify implementations by checking $Z \models_{\mathcal{L}} \Phi$ over Wymore execution traces (Figure~\ref{fig:comparison}B).

\miniheading{Contributions}
\begin{itemize}
    \item A parameterized assertional definition of the Functionality Cotyledon, $\mathcal{FC}_{\mathcal{L}}(\Phi) = \{ Z \mid Z \models_{\mathcal{L}} \Phi \}$, reframing functional verification as temporal property checking over Wymore trajectories rather than as construction of $Z_{\text{spec}}$ and $h$.
    \item Trace-based verification semantics aligned with Wymore's state- and output-trajectory functions ($\text{STZ}$, $\text{OTZ}$) \parencite{Wymore1993}.
    \item A workflow comparison of constructive versus assertional verification and an interpretive mapping to validation activities, demonstrated on a decentralized UAV swarm with discrete-time trajectories over infinite state (including continuous-valued components) and dynamic participation change.
\end{itemize}

The article proceeds as follows: Background summarizes T3SD and temporal logics; Methodology defines trace semantics, expressivity, and compares verification workflows; Results presents a UAV swarm case study; Discussion relates assertional FC to classical T3SD and validation practice.

\section{Background}

\subsection{T3SD}
Wymore's T3SD organizes system design into three cotyledons (design spaces): Functionality, Buildability, and Implementability \parencite{Wymore1993}. The Functionality cotyledon contains system designs that satisfy required input/output behavior and preferred performance ordering. Buildability captures technology constraints; Implementability is the space of designs that homomorphically map buildable artifacts to functional ones. In classical verification, functional intent is realized constructively: stakeholder requirements are formalized into a reference system $Z_{\text{spec}}$, and implementations are checked via morphisms (Figure~\ref{fig:comparison}A).

\subsection{Wymore Discrete Systems}
We restrict attention to Wymore's discrete systems (DSYSTEMS): a deterministic 5-tuple $Z = (S, I, O, \delta, \rho)$ with state set $S$, input set $I$, output set $O$, transition function $\delta$, and readout $\rho$ \parencite{Wymore1993}. Wymore explicitly allows $S$, $I$, and $O$ to be \emph{not-necessarily-finite} where finiteness is a practical special case. State components may include continuous-valued coordinates sampled at discrete, atomic ticks (e.g., $x,y \in \mathbb{R}$). Given an input trajectory $f$ over discrete time, Wymore's trajectory theorems yield unique state and output trajectories $g = \text{STZ}(f, s_0)$ and output trajectory functions. Engineered systems of interest are open: validation is performed at the system boundary through observable inputs and outputs, consistent with Systems Theory \parencite{salado2019mbse}.

\subsection{Temporal Logics}
Linear Temporal Logic (LTL) expresses requirements over infinite execution traces using operators such as $G$ (always), $F$ (eventually), $X$ (next), and $U$ (until) \parencite{Pnueli1977,BaierKatoen2008}. Safety properties (e.g., $G \neg \text{Hazard}$) forbid bad states; liveness properties (e.g., $G(\text{Trigger} \rightarrow F \text{Response})$) require progress. \textbf{First-Order LTL (FO-LTL)} extends propositional LTL with quantifiers over elements of $S$, $I$, or $O$ at each tick.  FO-LTL is needed over LTL when requirements relate continuous variables (e.g., $\Box\forall x \in \mathbb{R}\,(\text{Input}(x) \rightarrow \Diamond(\text{State} = \text{State}_{\text{prev}} + x))$). Signal Temporal Logic (STL) and Metric Temporal Logic (MTL) add continuous-signal and timed-interval constraints for cyber-physical systems \parencite{MalerNickovic2004}; TLA+ treats refinement as logical implication between temporal formulas \parencite{Lamport2002}. Functional intent in T3SD is inherently trajectory-based: a system design is acceptable when its behavior over time satisfies required transformations \parencite{salado2019mbse}.

\subsection{Related Formalisms}
Generic temporal model checking on finite automata does not, by itself, supply Wymore trajectory semantics, cotyledon interpretation, or a constructive-versus-assertional account of T3SD verification. Continuous-time hybrid verification (e.g., differential dynamic logic \parencite{platzer2018logical}) addresses dynamics \emph{between} discrete ticks; our framework evaluates properties at atomic Wymore clock steps. DEVS provides an event-scheduled simulation formalism complementary to Wymore's uniform time scale \parencite{wach2021devs}.

Assertional FC matters to MBSE practitioners as it connects checkable temporal requirements to Wymore's MBSE foundations and to validation workflows described in the INCOSE Handbook \parencite{INCOSE2023,salado2019mbse} without requiring engineers to maintain reference systems and homomorphisms as implementations evolve.

\section{Methodology}

\subsection{Verification Paradigm Comparison}
Figure~\ref{fig:comparison} contrasts Wymore's classical constructive verification with the proposed direct assertional verification. Under the assertional paradigm, functional intent is encoded as $\Phi$ in dialect $\mathcal{L}$ and checked by evaluating whether execution traces of $Z$ satisfy each property in $\Phi$ \parencite{BaierKatoen2008}.

Functional requirements in T3SD are naturally expressed as input/output behavior over time \parencite{Wymore1993,salado2019mbse}. When implementations change structure, maintaining universal constructive proofs over $Z_{\text{spec}}$ and $h$ can require substantial re-verification. Assertional checking instead evaluates whether the post-change system satisfies $\Phi$ on its trajectories, without neccesitating the construction of a new reference system.

\subsection{Formal Trace Definitions and Observability}
Given a Wymorean system $Z = (S, I, O, \delta, \rho)$, an initial state $s_0 \in S$, and a complete input trajectory $f \in \text{ITZ}$, Wymore defines a unique \textbf{state trajectory} $g \in \text{FNS}(\text{TZ}, S)$ and \textbf{output trajectory} $h \in \text{FNS}(\text{TZ}, O)$ \parencite{Wymore1993}. We write $g(t)$ for the state at discrete time $t$ and $h(t)$ for the output at $t$, where $\text{TZ}$ is Wymore's discrete time scale. Equivalently, $g = \text{STZ}(f, s_0)$ and the output trajectory follows Wymore's $\text{OTZ}$ construction. The recursive form below matches Wymore's construction:
\begin{align}
    g(0) &= s_0 \\
    g(t+1) &= \delta(g(t), f(t)) \\
    h(t) &= \rho(g(t))
\end{align}
We define the complete execution trace as the sequence $\sigma = (g(0), h(0)), (g(1), h(1)), \dots$. We consider temporal requirements $\Phi$ under observability constraints:
\begin{itemize}
    \item \textbf{External Observability (Black-Box):} Properties $\Phi_{\text{ext}}$ are defined only over output trajectories $h(t)$ and inputs $f(t)$, representing standard validation where internal states are hidden.
\end{itemize}
We say $Z \models_{\mathcal{L}} \Phi$ if, for all valid input trajectories $f$, the resulting trace satisfies $\sigma \models_{\mathcal{L}} \Phi$ under the semantics of the chosen dialect $\mathcal{L}$ (propositional LTL semantics as default notation) \parencite{Pnueli1977,BaierKatoen2008}. When $\mathcal{L}$ is clear from context we write $Z \models \Phi$. Assertional and classical FC need not coincide: a morphism guarantees structural simulation of internal transitions, while temporal properties on I/O traces may under-specify internal structure or over-specify behavior via liveness clauses. Section~\ref{sec:classical-fc} discusses this relationship.

\subsection{Theoretical Expressivity vs.\ Computational Tractability}
\label{sec:expressivity}

Classical T3SD verification asserts \textbf{language inclusion}: every input/output trajectory of $Z_{\text{impl}}$ must be a valid trajectory within the universe defined by $Z_{\text{spec}}$. With infinite $S$ and arbitrary $\delta$, $Z_{\text{spec}}$ can encode rich behaviors; assertional FC must not cap that richness at propositional LTL, which is \emph{strictly less expressive} than Wymore morphisms (finite atomic propositions cannot state $\forall x \in \mathbb{R}$ input-output relations).

\textbf{Theoretical definition.} To preserve unconstrained specification potential, $\mathcal{FC}_{\mathcal{L}}(\Phi)$ admits a spectrum of temporal logics; FO-LTL suffices for trajectory-level specs over infinite domains at ticks, and MSO-extended FO-LTL provides a theoretical ceiling for expressing structural existence claims (morphism existence) \parencite{Thomas1990}. We do not claim a completed equivalence proof between MSO/FO-LTL and classical morphisms; the expressivity hierarchy is an honest design commitment, not an expressive restriction on the cotyledon.

\textbf{Paradigm shift.} Wymore used morphisms because declarative temporal logics were not the MBSE lingua franca in 1993---not because engineers sought uncomputable $Z_{\text{spec}}$ artifacts for their own sake. Declarative $\Phi$ in a sufficiently rich $\mathcal{L}$ can state the same trajectory constraints without building $Z_{\text{spec}}$.

\textbf{Pragmatic restriction.} While theory admits FO-LTL/MSO, model authors and tools may restrict $\mathcal{L}$ to propositional LTL, bounded FO-LTL, or STL fragments for decidable automated checking (Lean, finite-state model checking, tick-local SMT). This is an \emph{authoring choice}, not a cotyledon definition.

\subsection{Discrete Trajectories and Verification Bounds}
This article scopes verification to Wymore DSYSTEMS: trajectories evolve at discrete time steps according to $\delta$ and $\rho$, and $S$, $I$, and $O$ may be infinite \parencite{Wymore1993}. The satisfaction relation $Z \models_{\mathcal{L}} \Phi$ is defined over these trajectories regardless of cardinality. What is out of scope is \textbf{continuous-time} evolution between ticks (formal verification via dL or hybrid automata); continuous-\emph{valued} state at ticks is in scope.

When $|S|$, $|I|$, and $|O|$ are finite, standard finite-state model checking applies \parencite{BaierKatoen2008}. When one or more sets is infinite, automated checking requires abstraction, invariant generation, bounded prefix analysis, or theorem proving; decidability depends on the property fragment and domain. The swarm case study uses tick-local SMT evaluation as an engineering tactic under FO-LTL, not as a universal method (Section~\ref{sec:case-study}).

\section{Results}

\subsection{Case Study: Decentralized UAV Swarm}
\label{sec:case-study}

We evaluate the assertional framework on a decentralized swarm of $N$ UAVs executing cooperative search over a bounded region $R \subset \mathbb{R}^2$ while maintaining minimum separation $d_{\text{safe}}$. Each UAV is a Wymore subsystem $Z_i$; the aggregate state is the product of individual states sampled at discrete flight-controller ticks (e.g., 10~Hz):
\[
S_i = (x_i, y_i, \psi_i, v_i, b_i) \in \mathbb{R}^4 \times [0, 100].
\]
Because coordinates and velocities are real-valued, $|S_i|$ and the product $|S_{\text{swarm}}|$ are infinite. Wymore specifies such systems via $(S,I,O,\delta,\rho)$; the verification burden is proving universal properties over trajectories, not writing $\delta$ itself.

\miniheading{System Model and RTB}
\textbf{Return to Base (RTB)} is a local fault-response mode: when battery $b_i$ falls below a critical threshold, UAV$_i$ autonomously exits cooperative swarm mission, navigates to base, and becomes \textbf{inactive} in the swarm \emph{active mask} $M \subseteq \{1,\ldots,N\}$---a transition on participation encoded in $\delta$, not an inability to specify the system classically.

\miniheading{Constructive Baseline: Parameterized Swarm}
Classical T3SD can represent participation via system parameterization (Def.~2.82) and conjunctive coupling (CSY) over $N_{\max}$ slots, with mask $M$ carried in global state. A naive approach---one $Z_{\text{spec}}$ and morphism $h_r$ per active subset---scales as $2^N$ instances. A unified model with mask in state avoids exponential \emph{system definition} count; universal safety/liveness proofs over all valid masks remain non-trivial. We formalize mask-based participation for $N \in \{1,2,3\}$ in Lean~4 (\texttt{SwarmExamples.lean}): trajectory uniqueness at $N=1$; \texttt{ActiveAgents} and \texttt{SwarmSeparation} use \texttt{Finset} quantifiers over active indices rather than $2^N$ mask case splits; RTB drops agents from the active set.

\miniheading{Discrete-Tick Kinematics and Verification Tactic}
Transitions follow $\delta$ at discrete ticks; between ticks, kinematics evolve continuously in operation, but verification evaluates separation only at sampled states:
\[
(x_i(t+1), y_i(t+1)) = f_{\text{kin}}(x_i(t), y_i(t), \delta_{\text{cmd}}(t)).
\]
For collision avoidance at each tick, an SMT solver checks pairwise distance constraints over the recorded state vector. This is a \emph{case-study-specific} tactic: it under-approximates inter-tick motion and does not constitute complete continuous-time verification.

\miniheading{Functionality Cotyledon as $\Phi$ (FO-LTL)}
We instantiate dialect $\mathcal{L}$ as \textbf{FO-LTL} for this example; propositional LTL would require finite abstraction of coordinates into atomic propositions. Functional intent is:
\begin{align*}
\varphi_{\text{safe}} &= G \bigl( \forall i,j \in N,\, i \neq j \rightarrow d_{ij} \geq d_{\text{safe}} \bigr) \\
\varphi_{\text{live}} &= G \bigl( \text{Command}_{\text{search}} \rightarrow F\, \text{Coverage}(R) = 100\% \bigr)
\end{align*}
where $d_{ij} = \sqrt{(x_i - x_j)^2 + (y_i - y_j)^2}$ and $\Phi = \{\varphi_{\text{safe}}, \varphi_{\text{live}}\}$. Quantifiers range over agents and real-valued distances; STL would offer an alternative CPS-oriented formulation for dense-time signal bounds.

\miniheading{Requirement Encoding Example}
\begin{description}
    \item[Natural language:] Each UAV shall maintain at least $d_{\text{safe}}$ from every other UAV at all times.
    \item[FO-LTL:] $\varphi_{\text{safe}}$ above (instantiated for swarm index set $N$).
\end{description}

\miniheading{Illustrative Instance: RTB ($N=10 \rightarrow 9$)}
Suppose UAV$_3$ enters RTB after a battery fault; agent~3 becomes inactive in $M$ while nine agents continue. \textbf{Constructive:} unless a universal invariant over mask transitions was proved upfront, RTB triggers a new proof obligation for the mask change. \textbf{Assertional:} the same $\Phi$ is re-checked on trajectories of the reduced active set; if $\varphi_{\text{safe}}$ and $\varphi_{\text{live}}$ hold (quantifying only over active agents), then $Z_{\text{impl}} \models_{\mathcal{L}} \Phi$. Lean~3 confirms RTB as a mask transition preserving separation when only one agent remains active.

\miniheading{Formal Exploration (Lean 4)}
For $N=1$, a single \texttt{DiscreteSystem} yields trajectory uniqueness. For $N=2$ and $N=3$, product-state steps with \texttt{ActiveAgents} and \texttt{SwarmSeparation} demonstrate RTB as a mask transition and indexed separation invariants without $2^N$ case analysis. Universal constructive proof over all masks and all real trajectories at scale remains open.

\miniheading{Evaluation Rubric (Instantiated)}
The swarm exhibits: (1) concurrent subsystems with infinite product state; (2) continuous-valued state at discrete ticks; (3) FO-LTL safety and liveness in $\Phi$; and (4) participation change under RTB as proof-maintenance, not impossibility of classical T3SD. We do not report empirical timing benchmarks; complexity is argued qualitatively from coupling structure and mask-based proof patterns.

\section{Discussion}

Encoding functional requirements as temporal logic moves intent from natural-language statements into a checkable formal language \parencite{salado2019mbse,BaierKatoen2008}, supporting MBSE validation by making temporal requirements executable against Wymore trajectory semantics. A second benefit is decoupling intent from implementation structure: when $\delta$ or $\rho$ change, engineers re-check $Z \models_{\mathcal{L}} \Phi$ rather than rebuilding homomorphisms.

\subsection{Relationship to Classical Functionality Cotyledon}
\label{sec:classical-fc}

Classical T3SD defines functional designs constructively; assertional FC defines them by temporal properties in dialect $\mathcal{L}$. These are not equivalent in general. A system homomorphism $h$ asserts language inclusion: trajectories of $Z_{\text{impl}}$ map into valid trajectories of $Z_{\text{spec}}$. Morphism existence implies trace inclusion under the homomorphic map---one direction of relationship. Conversely, $Z \models_{\mathcal{L}} \Phi$ does not imply existence of $h$ to a given $Z_{\text{spec}}$, and propositional LTL \emph{under-specifies} relative to morphisms; FO-LTL and MSO-extended FO-LTL at the theoretical ceiling aim for parity without shrinking the specification universe. A full characterization of when $\mathcal{FC}_{\mathcal{L}}(\Phi)$ aligns with the classical solution space remains open.

Constructive verification can still be preferable when $|S|$ is small, $Z_{\text{spec}}$ already exists, and changes are internal refinements with fixed observable behavior (e.g., state renaming). In such cases, updating $h$ may be simpler than revising $\Phi$.

\subsection{Limitations and Open Questions}
\label{sec:limitations}

\begin{itemize}
    \item \textbf{Equivalence:} Assertional and classical FC are not proven equivalent; Table~\ref{tab:comparison} compares workflows, not logical strength. MSO/FO-LTL $\leftrightarrow$ morphism equivalence is open.
    \item \textbf{Theory vs.\ tooling:} Theory admits FO-LTL/MSO; propositional LTL is insufficient for the swarm without FO quantifiers. Tooling may restrict $\mathcal{L}$; FO-LTL over reals is undecidable in general.
    \item \textbf{Infinite state:} Tick-local SMT checks are an under-approximation for inter-tick motion; continuous-time between ticks is out of scope.
    \item \textbf{Constructive tractability:} Unified mask state avoids $2^N$ system definitions; universal proofs over all masks/reals remain open (Lean $N \leq 3$ exploration).
    \item \textbf{Scope:} General contribution is trace semantics plus parameterized temporal logics; SMT-at-ticks is swarm-specific.
    \item \textbf{Autonomous steps:} Lean encoding uses \texttt{Option} for empty inputs \parencite{demoura21lean4}; liveness over autonomous (\texttt{none}) steps---formal treatment deferred.
    \item \textbf{Empirical evidence:} No benchmark comparing morphism-construction effort to property checking; swarm evidence is qualitative.
\end{itemize}

\section{Conclusions}

The objective of this work is to reinterpret Wymore's Functionality Cotyledon assertional rather than constructive. We define $\mathcal{FC}_{\mathcal{L}}(\Phi) = \{ Z \mid Z \models_{\mathcal{L}} \Phi \}$, ground satisfaction in Wymore's $\text{STZ}$/$\text{OTZ}$ trajectories, and compare verification workflows against classical T3SD. The parameterized temporal-logic framework admits FO-LTL and MSO at the theoretical ceiling without shrinking classical expressivity; practitioners may restrict $\mathcal{L}$ for tractable tooling. The decentralized UAV swarm illustrates discrete-time trajectories over infinite state (including continuous-valued components), FO-LTL safety/liveness requirements, and verification across dynamic participation change without rebuilding $Z_{\text{spec}}$ and $h$. Future work includes formal relationships between assertional and classical FC, complete autonomous-step liveness semantics, and tool-chain integration aligned with INCOSE validation practice \parencite{INCOSE2023}.
